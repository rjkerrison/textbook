Pong

Building a pong game is complicated. There are a lot of steps, and we have to use HTML5 Canvas, which is an advanced topic.

If we break it down, a simplified version of a pong game might be one where the paddles don't move. We could start just by drawing a single frame of a pong game.

We can build a pong game in a series of steps, each one bringing us closer to the final result.

First step: defining paddles

Prerequisites: Defining Paddles

When experimenting, this wasn't the first place I looked. I started with building a static image of a pong game. As I extended that, it became clear that I needed to store paddle information in a reliable way. That means making a class Paddle {}.

We need to be able to store paddles and draw them. They'll have a position, a configurable colour, and we also need to have a way of checking if the ball is in the horizontal range covered by the paddle.
